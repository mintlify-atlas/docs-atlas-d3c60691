## Endpoint

```
GET /deviantart/get-all-media
```

## Parameters

<ParamField query="username" type="string" required>
  DeviantArt username. The leading `@` is stripped automatically if present.
</ParamField>

<ParamField query="limit" type="number">
  Maximum number of items to download. Set to `0` or omit for no limit. The count includes the profile header and avatar.
</ParamField>

## Example

```bash
curl "http://localhost:3000/deviantart/get-all-media?username=Loish&limit=50"
```

## How it works

The DeviantArt engine uses a hybrid Puppeteer/REST approach:

1. **Authentication** — PerimeterX cookies (`_px`, `_pxvid`, `pxcts`) are injected from environment variables. A real-time network interception captures the `csrf_token` from the first authenticated request.
2. **Profile metadata** — The profile header (banner) is extracted via computed CSS evaluation on the DOM. The avatar URL is retrieved from the API `usericon` field.
3. **Gallery fetch** — When authenticated, the controller calls the internal DeviantArt API to collect all gallery item URLs. When unauthenticated (cookie bypass failed), it falls back to HTML scraping.
4. **Smart fallback URLs** — If the primary download token (`token[1]`) is absent, the engine constructs a fullview URL from the view token (`token[0]`) to preserve maximum image quality.
5. **File extension mapping** — The correct extension (`.jpg`, `.png`, etc.) is determined from the `filetype` field in the API response, not guessed from the URL.
6. **Downloads** — All tasks are executed concurrently via `mapLimit` with a random 500–1500 ms jitter between each file to avoid rate limiting.

## Environment variables

<Warning>
  The DeviantArt engine requires all six PerimeterX-related cookies to be present. Without them, requests will be blocked before any gallery data is returned.
</Warning>

| Variable | Cookie name | Description |
|---|---|---|
| `DA_AUTH` | `auth` | Main authentication cookie |
| `DA_AUTH_SECURE` | `auth_secure` | Secure auth cookie |
| `DA_USERINFO` | `userinfo` | User session info |
| `DA_PX` | `_px` | PerimeterX long-lived token |
| `DA_PXVID` | `_pxvid` | PerimeterX visitor ID |
| `DA_PXCTS` | `pxcts` | PerimeterX timestamp |

Add these to your `.env` file:

```env
DA_AUTH=your_auth_cookie_value
DA_AUTH_SECURE=your_auth_secure_value
DA_USERINFO=your_userinfo_value
DA_PX=your_px_value
DA_PXVID=your_pxvid_value
DA_PXCTS=your_pxcts_value
```
